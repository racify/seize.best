Security & Antinuke

The Seize Antinuke System is designed to protect your server from malicious administrators or compromised staff accounts. It monitors high-impact actions in real-time, automatically revoking permissions or taking punitive actions if threshold limits are exceeded.

Monitored Violations

When enabled, the antinuke system monitors and protects against the following actions:

  • Excessive Channel Edits: Rapidly creating or deleting server channels.
  • Excessive Role Edits: Rapidly creating, deleting, or editing server roles.
  • Mass Member Punishments: Banning or kicking multiple members in a short timeframe.
  • Unauthorized Bot Additions: Automatically blocks and kicks any external bot added to the server by non-whitelisted administrators.

Setting Up the Antinuke System

Follow these steps to configure and enable your server defenses:

  1. Enable Antinuke: Toggle the global security system on. 
    ,antinuke toggle
  2. Whitelist Trusted Administrators: Add co-owners or highly trusted staff to the whitelist so they are exempt from antinuke checks. 
    ,antinuke whitelist add @User
  3. Configure Action Limits: Define the thresholds for how many edits are allowed before the bot triggers. 
    ,antinuke limits
Warning

The antinuke system is highly sensitive and aggressive to ensure maximum security. Please ensure all key administrators are added to the whitelist using ,antinuke whitelist add immediately upon enabling the system to prevent accidental locks or bans.

Moderation & Jail Rooms

Seize provides server staff with professional, reliable moderation utilities. In addition to standard moderation commands like ,ban, ,kick, ,mute, and ,warn, Seize features a unique, fully-integrated Jail System.

The Jail Isolation System

When a user is jailed, Seize strips them of all their current roles (saving them in the database) and applies a custom Jailed role. They are immediately locked in a dedicated, isolated channel where they can only speak to administrators to explain their actions or read server rules. Once unjailed, their original roles are automatically restored.

Setting Up the Jail System

Setting up and using jail rooms is simple and takes only two steps:

  1. Run the Setup Command: This instructs Seize to automatically create a secure #jail channel and the restricted Jailed role with proper permission overrides configured across all existing channels. 
    ,jail setup
  2. Jailing a User: Put a member in jail by specifying their name and an optional reason. 
    ,jail @Member breaking rules
  3. Releasing a User: Restore their roles and release them from the jail cell. 
    ,unjail @Member

Voicemaster (Dynamic VCs)

The Voicemaster system allows server members to create and manage temporary voice channels dynamically. When a user joins a designated "Join to Create" channel, Seize instantly creates a new, private voice channel for them under a custom category. The creator becomes the owner and can lock, unlock, rename, or limit the channel. When the last user leaves, the channel is deleted automatically.

Voicemaster Setup Guide

Follow these steps to configure dynamic voice channels:

  1. Run the Setup Command: This creates a custom Voicemaster category, a "Join to Create" voice channel, and a text channel containing an interactive button control panel. 
    ,voicemaster setup
  2. Create Your Channel: Join the newly created "Join to Create" voice channel. You will be instantly moved to your own private channel.
  3. Manage with Commands: Use the following commands to customize your voice channel on the fly:

    Lock/Unlock: ,vc lock / ,vc unlock (Prevent or allow users to join)

    Rename Channel: ,vc rename [new name] (Change the name of your channel)

    Set Limit: ,vc limit [number] (Set the maximum user cap)

    Permit User: ,vc permit @User (Let a specific user join your locked channel)

    Reject/Kick User: ,vc reject @User (Kick a user out and block them)

    Claim Channel: ,vc claim (Claim ownership if the original creator left)

Earning and Saving Coins

Use the following commands to manage your wallet and bank balance:

Tickets & Automations

Seize provides high-fidelity automated tools to run your community professionally, including an advanced Support Ticket System, custom greeting/welcome cards, vanity role trackers, and autoroles.

Setting Up the Ticket System

The ticket system creates beautiful button panels that members click to open private channels for support. Staff can manage, claim, and close tickets directly using button interfaces. Follow these steps to initiate ticket setups:

  1. Initialize Ticket Setup: Run the interactive setup command. 
    ,ticket setup
  2. Configure Panels: Follow the interactive prompts in Discord to select the support roles, customize the message embeds, and designate the target channel where the ticket panel should be deployed.
  3. Manage Ticket Channels: Once a member opens a ticket, support staff can resolve their issue and archive/close the ticket channel using the panel buttons or commands: 
    ,ticket close

Server Welcome & Leave Messages

Welcome new members with custom embed greetings and leave alerts:

  • Set Welcome Channel: ,welcome channel #welcome
  • Set Welcome Message: ,welcome message [embed JSON or text]
  • Set Leave Channel: ,leave channel #goodbyes
